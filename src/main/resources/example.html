Web‑Beispiele für die API

Diese Seite zeigt reine Web‑Beispiele (keine curl-Beispiele). Jede Aktion zeigt vor dem Senden eine Vorhersage (Predicted Status + Begründung) und nach der Antwort den tatsächlichen Status (Actual) sowie die Ableitung.

GET /user?id=...

Gib eine ID ein und sende das Formular (GET). Die Seite fragt vorab /users ab, um vorhandene IDs zu kennen und eine Vorhersage zu treffen.

Noch keine Anfrage.

POST /user (HTML‑Form oder JSON)

Erstelle einen neuen Benutzer. Wähle das Format (Form oder JSON) und „Senden (inline)" um Vorhersage + Antwort hier zu sehen.

Noch keine Anfrage.

PUT /user/{id}, PATCH /user/{id}, DELETE /user/{id} (Web‑Buttons)

Die folgenden Buttons verwenden Fetch‑API. Vor dem Senden wird /users abgefragt, um die Vorhersage zu erstellen.

Keine Aktion ausgeführt.
